# Business Events Collector
Сервис для обработки файлов с бизнес-событиями и отправки событий в систему аналитики.

## Описание
Основная задача сервиса — чтение файлов событий, десериализация записей в объекты и отправка событий в очередь сообщений.
Сервис запускается на конечном устройстве (например, кассовом аппарате) и обрабатывает файлы с событиями операций. 
Файлы содержат данные о чеках, которые были оплачены или отменены.

Проект является частью системы аналитики бизнес-событий и включает два компонента:
- **Collector** — запускает обработку файлов с событиями, используя Processor для параллельной обработки и отправки данных.
- **Generator** — инструмент для создания тестовых файлов с событиями.

## Используемые библиотеки
- **Moq / XUnit** — для модульного тестирования
- **Serilog** — для логирования
- **Bogus** — для генерации тестовых данных

## Структура решения
Решение состоит из нескольких проектов:

###  dnaborshchikova_github.Bea.Collector.App
Консольное приложение для разового запуска обработки файлов из заданной директории. 
Запускает обработку файлов с событиями, используя Processor для параллельной обработки и отправки данных. 
Читает один файл и завершает работу.

### dnaborshchikova_github.Bea.Collector.WorkerService
Сервисное приложение, которое запускает обработку файлов с событиями через Processor.
Работает в двух режимах:
- Однократная обработка файлов (например, при запуске в Docker-контейнере)  
- Периодический запуск (раз в 24 часа) как воркер-сервис ОС.

### dnaborshchikova_github.Bea.Collector.Core
Содержит базовые модели и интерфейсы проекта.

### dnaborshchikova_github.Bea.Collector.DataAccess
Проект содержит репозиторий и контекст базы данных. Отвечает за инициализацию базы данных и сохранение событий.

### dnaborshchikova_github.Bea.Collector.Parser
Модуль разбора файла событий и десериализации данных в объекты.

### dnaborshchikova_github.Bea.Collector.Processor
Центральный компонент обработки событий.  
- Получает файлы с событиями от App или WorkerService.  
- Разбирает файлы и преобразует записи в объекты событий.  
- Делит события на части для параллельной обработки (потоки или задачи в зависимости от настроек).  
- Отправляет события в базу данных или в очередь сообщений.  
- Логирует результаты обработки каждого файла.

### dnaborshchikova_github.Bea.Collector.Sender
Отправляет события в очередь сообщений и базу данных.

### dnaborshchikova_github.Bea.Collector.Tests
Модульные тесты для проекта.

### dnaborshchikova_github.Bea.Generator
Генератор тестовых файлов с событиями для проверки работы системы.

## Поддерживаемый формат файлов
Файлы с событиями должны иметь расширение **CSV**. Поддерживаются два типа событий:

### Событие оплаты чека (`bill_payed`) //точно или в этом месте может ниже где запуск?

| Поле | Описание |
|-----|-----|
| OperationDateTime | дата и время операции |
| UserId | GUID кассира |
| EventType | `bill_payed` |
| Amount | сумма чека |
| Number | номер чека (6-значный, уникален в течение дня) |
| Id | GUID чека |
| BuyerId | GUID покупателя (необязательный) |

### Событие отмены чека (`bill_canceled`)

| Поле | Описание |
|-----|-----|
| OperationDateTime | дата и время операции |
| UserId | GUID кассира |
| EventType | `bill_canceled` |
| Amount | общая сумма чека |
| CancelAmount | сумма отмены |
| Number | номер чека (6-значный, уникален в течение дня) |
| Id | GUID чека |

### Настройки

Сервис поддерживает конфигурацию через `appsettings.json`.  
Основные настройки:

- **ProcessingSettings** — параметры обработки файлов:
  - `ThreadCount` — количество потоков для параллельной обработки событий.
  - `ProcessType` — способ обработки (`Thread` или `Task`).
  - `FilePath` — путь к обрабатываемому файлу.
  - `GeneratorRunAsProcess` и `GenerateFile` — позволяют запускать генератор тестовых файлов прямо через консольное приложение.
- **GeneratorSettings** — параметры генерации файлов:
  - `FileFormat` — формат файла (`csv`).
  - Количество событий (`PaidBillEventCount`, `CancelledBillEventCount`) для тестирования больших файлов.

> Processor автоматически делит файл на диапазоны по датам и обрабатывает события параллельно в соответствии с настройками.
